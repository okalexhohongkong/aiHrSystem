# 黑卫士 AI HR 系统 V1.2 可演示版

这是黑卫士 AI HR 系统的 Web/PWA 演示版本，当前目标是用于本机安装、页面演示和业务流程确认。它不是最终生产版，真实数据库、真实短信/邮件/微信发送、真实招聘平台接口、真实 ASR/AI 接口会在单机可用版继续接入。

## 当前可演示范围

- 总览看板：候选人总数、推荐复试、录音解析、待提交作业、岗位滚动窗口。
- 候选看板：候选人长条列表、投递时间排序、匹配度、资料完整度、同行权重、技能标签。
- 岗位发布：AI 辅助 JD、岗位类型、岗位动态数据、标杆公司画像。
- 招聘账号：多招号招聘矩阵、登录方式预留、招聘平台适配器预留。
- 邀约预约：邮件、电话、短信、微信、企微、平台留言预约流程。
- 录音解析：录音上传、会议接入、转写纪要、面试档案分析的前端流程。
- 初试评分：九维模型、可拖拽维度、蜘蛛网和长条评分表达。
- 岗位问卷：岗位问卷卡片库、题库、加分项、减分项、AI 反向生成问卷。
- AI 邀约面试：文字/语音 AI 面试流程、佐证材料要求。
- 面试编排：在线、初试、复试、终试的轮次安排。
- 三轮作业：作业题、出题人、考官、答卷、评分、案例归档。
- 自动分工：自动分配人事/面试官、双向确认、超时报警。
- 端口 Agent：Agent Skills、APP/PWA 下载通道、短信/邮件/二维码/链接发送入口。
- 简历库、数据看板、成果目标、合约模块、反向求职调研、合伙兼职、移动工作端、平台底座、多端路线、风险合规。
- 界面皮肤：企业 Logo/名称、颜色调色盘、字体、字号、左侧菜单宽度/字号/行距、母菜单显示隐藏。
- 系统后台：账号、权限、版本等级、招聘账号连接管理。演示权限码：`123456`。

## 从 GitHub 下载后启动

推荐使用固定依赖安装命令，确保别人下载后界面和当前版本一致：

```bash
git clone git@github.com:okalexhohongkong/aiHrSystem.git
cd aiHrSystem
npm ci
npm run demo
```

然后打开：

```text
http://127.0.0.1:5173/
```

如果已经安装过依赖，以后只需要在项目目录执行：

```bash
npm run demo
```

也可以双击项目根目录的 `start-demo-mac.command` 启动。

## 生产演示包

生成静态演示包：

```bash
npm run build
```

构建产物在：

```text
dist/
```

本机预览生产包：

```bash
npm run demo:prod
```

## Mac 单机 App 打包

生成经典版和国际版两个 Mac 单机演示包：

```bash
./scripts/package-mac-demo.sh all
```

生成后的位置：

```text
release/黑卫士 AI HR 经典版-mac.dmg
release/黑卫士 AI HR 经典版-mac.zip
release/黑卫士 AI HR 国际版-mac.dmg
release/黑卫士 AI HR 国际版-mac.zip
```

说明：

- 经典版是 A 计划主线，不显式展示国际 HR 参考库。
- 国际版是 B 计划备用，显式展示国际 HR 参考库。
- `release/` 和 `dist/` 是本机生成产物，不提交到 Git；别人克隆后用上面的命令重新生成即可。

## 安装到 Mac 桌面

当前版本是 Web/PWA，可通过浏览器安装到桌面：

1. 先启动 `npm run demo` 或 `npm run demo:prod`。
2. 用 Chrome 或 Edge 打开 `http://127.0.0.1:5173/`。
3. 点击地址栏右侧的“安装应用”图标。
4. 安装后可以像普通桌面应用一样从启动台打开。

Safari 可使用“文件 / 添加到程序坞”方式安装为桌面入口。

## 演示检查顺序

1. 打开总览，确认岗位与面试者滚动窗口正常显示。
2. 点候选看板，确认长条候选人列表、排序按钮、匹配度和标签可见。
3. 点岗位发布，确认 JD 生成、标杆公司画像、岗位动态数字可见。
4. 点界面皮肤，测试企业 Logo、颜色、字体和字号设置。
5. 点系统后台，输入演示权限码 `123456`，确认权限和账号管理页面可见。
6. 点端口 Agent，确认 APP 下载、二维码、短信/邮件/链接通道合并入口可见。
7. 点三轮作业、自动分工、AI 邀约面试，确认核心招聘流程页面可见。

## 保证界面一致的要求

- 使用 `npm ci`，不要用不同版本依赖覆盖 `package-lock.json`。
- 提交 `src/`、`public/`、`package.json`、`package-lock.json`、`packaging/`、`scripts/`。
- 不提交 `node_modules/`、`dist/`、`release/`。
- 克隆后用 `npm run demo` 打开 `http://127.0.0.1:5173/`。
